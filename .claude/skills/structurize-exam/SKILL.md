# 過去問OCR → 構造化JSON変換スキル

## 概要

応用情報技術者試験（AP）午前問題のOCRテキストを読み、構造化されたJSONデータに変換する。
正規表現やスクリプトベースの処理は行わず、LLMとしてテキストを読解して構造化する。

## スキーマ

出力JSONのスキーマは `references/schema.md` に定義されている。作業前に必ず読むこと。

## ワークフロー

### Phase 1: 入力データの確認

1. `ref/past_exams_ocr/` 配下のOCRテキストファイル一覧を取得する
   - ファイル名パターン: `{exam_id}_ap_am_qs.txt`（33ファイル）
2. `ref/past_exams_structured/` の既存JSONを確認し、未処理・要再処理のファイルを特定する
3. 処理対象を決定する（ユーザーに確認。1ファイルずつか、全件一括か）

### Phase 2: OCRテキストの読解と構造化（1試験分）

対象ファイルを1つずつ処理する。以下の手順で進める。

#### Step 2-1: OCRテキストの読み込み

`ref/past_exams_ocr/{exam_id}_ap_am_qs.txt` を読む。

テキストの構造:
- `--- page-NN ---` で区切られたページ単位
- 最初の1〜3ページは表紙・注意事項・表記ルール（無視する）
- 問題は `問N` で始まる（N=1〜80）
- 選択肢は `ア` `イ` `ウ` `エ` で始まる
- 1ページに1〜2問が含まれることが多い

#### Step 2-2: 試験メタデータの抽出

1ページ目から `exam_name` を抽出する（例: 「令和7年度春期」）。
`exam_id` はファイル名から導出する。

#### Step 2-3: 問題の分割と構造化

OCRテキストを読み、問1〜問80を順番に特定して構造化する。

各問題について:
1. **問題文（body）**: `問N` の後から選択肢の前までがbody。OCRの誤読を文脈から判断して修正する。
2. **選択肢（choices）**: `ア` `イ` `ウ` `エ` で始まる部分を4択として抽出する。
3. **図の判定（has_figure）**: 以下のいずれかに該当する場合 `true`:
   - 問題文に「図」「表」「グラフ」等の参照がある
   - 選択肢がOCRテキストで判読不能（空白や断片的な文字のみ）
   - ページ内の文字量が明らかに少ない（図が占めているため）
4. **図の説明（figure_description）**: `has_figure: true` の場合、OCRテキストから推測できる範囲で図の内容を簡潔に説明する。

OCRテキストは完全ではない。特に:
- 数式・論理式の記号が化けていることが多い（可能な範囲で修正）
- 図を含む問題はテキストが断片的（`has_figure: true` にして補足説明をつける）
- 問題番号と問題文が連結していることがある（例: `問1論理式P,Q...`）

#### Step 2-4: 正解の取得

`ref/past_exams/{exam_id}_ap_am_ans.pdf` を読み、問1〜問80の正解を取得する。
解答PDFは通常1ページの表形式で、問題番号と正解（ア〜エ）が対応している。

pypdfium2 でテキスト抽出できない画像ベースのPDFもある。その場合:
- PDFをRead toolで画像として読み、目視で正解を確認する
- またはユーザーに正解の提供を依頼する

#### Step 2-5: タグ付け

各問題の内容を読み、`references/tags.md` に定義されたタグを付与する。
タグ付けの判断基準:

1. 問題文を読んで「何の知識を問うているか」を理解する
2. 該当するテキスト目次の節を特定する
3. 複数分野にまたがる問題は複数タグを付ける
4. 確信が持てない場合は空配列にする

AP午前問題の出題傾向（タグ付けの参考）:
- 問1〜問16あたり: 基礎理論（第1章）、アルゴリズム（第2章）
- 問17〜問24あたり: ハードウェア（第3章）、システム構成（第4章）
- 問25〜問32あたり: ソフトウェア（第5章）
- 問33〜問40あたり: データベース（第6章）
- 問41〜問48あたり: ネットワーク（第7章）
- 問49〜問60あたり: セキュリティ（第8章）
- 問61〜問68あたり: システム開発（第9章）、マネジメント（第10章）
- 問69〜問80あたり: ストラテジ（第11章）

ただし、これはあくまで傾向であり、問題の内容を読んで判断すること。

### Phase 3: JSON出力

構造化した結果を `ref/past_exams_structured/{exam_id}_am.json` に保存する。

保存前に以下を確認:
- questions 配列が80件あるか（不足の場合は理由をコメント）
- 全問に answer が設定されているか
- choices が4つ揃っていない問題は `has_figure: true` になっているか
- tags が付与されている問題の割合

### Phase 4: 全結合（全試験分が完了した後）

全33ファイルの処理が完了したら `ref/past_exams_structured/all_am.json` に結合する。
スキーマは `references/schema.md` の「全結合ファイル」セクションを参照。

## 注意事項

- OCRテキストの文字量は1ファイルあたり数千行に及ぶ。Read tool で分割読み込みし、問題単位で処理する。
- 1回の呼び出しで全33ファイルを処理するのは現実的ではない。ユーザーと相談して処理単位を決める。
- 既存の `ref/past_exams_structured/` にスクリプトで生成した低品質なJSONがある場合、上書きしてよい。
