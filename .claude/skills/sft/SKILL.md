# sft

応用情報技術者試験の Layer 2（SFT）実装。Layer 1（Pre-training）で作った知識表現を、**試験特有の問い方で確実に取り出せるように訓練する**。

## Layer 1 との違い

| | Layer 1 (Pre-training) | Layer 2 (SFT) |
|---|---|---|
| 目的 | 概念表現を脳に作る | 持っている知識を試験形式で出力する練習 |
| 問題の素材 | AI がテキスト知識から新規生成 | **過去問を seed にした変形問題をリアルタイム生成** |
| ペース | 1 leaf に15分、15問前後 | 1問 3〜5分、数十問を高速に回す |
| Engagement の所在 | 問題を解く行為自体 | **フィードバック側のメタ認知訓練** |

## 入力

learner の曖昧な指定をそのまま受け取る（Layer 1 と同じ）:
- leaf ID、節番号、タイトルの一部、章名

## 処理フロー

### Step 1: leaf を確定する（textbook-dispenser）

```bash
python3 /Users/tskymmt/git/aitee_training/.claude/skills/textbook-dispenser/scripts/dispense.py resolve "<learner input>"
```

ambiguous なら候補を提示して選んでもらう。Layer 1 と同じ。

### Step 2: 過去問 seed を取得する（past-exam-dispenser）

確定した leaf ID から section ID を抽出し（`ch04_03_01` → `ch04_03`）、そのセクションの過去問を全て取得する:

```bash
python3 /Users/tskymmt/git/aitee_training/.claude/skills/past-exam-dispenser/scripts/dispense.py by-section <section_id> --full
```

返ってきた問題群の中から、**対象 leaf に関連性が高いものを自分で選別**する。セクション内には複数 leaf の問題が混在しているため、leaf のタイトル・概念と照合して seed を選ぶ。

seed が 0 件の場合（稀）: 隣接セクションの過去問を参照するか、Layer 1 的な engagement 問題で代替する旨を learner に伝えてから進む。

### Step 3: Entry ゲート確認

理想は Layer 1 で全 leaves を完了してから Layer 2 に入ること。learner にその旨を軽く伝えるが、**強制はしない**:

> 「この leaf の Layer 1（pre-training）はお済みですか？ まだであれば先に概念を頭に入れてからの方が効果的ですが、このまま進めることもできます。」

learner が「進める」と言ったら進める。ゲーティングは soft。

### Step 4: 変形問題の生成と出題（メインループ）

seed 過去問をコンテクストに持ちながら、**1問ずつ変形問題をリアルタイム生成**して出題する。

#### 変形の方法

seed 問題を以下のいずれかの手法で変形する。同じ手法を連続させず、バラけさせる:

- **数値変更**: 稼働率の値、処理時間、台数など数値パラメータを差し替え
- **シナリオ差替**: 同じ概念を別の業種・場面に置き換え（銀行→病院、サーバ→ネットワーク機器）
- **正答/distractor 入替**: 元の正答を distractor に回し、別の切り口から正答を作る
- **問い方の逆転**: 「正しいものはどれか」→「誤っているものはどれか」
- **隣接概念との合成**: 同セクション内の別 leaf の概念と組み合わせた問題
- **抽象度の上下**: より具体的な事例問題にするか、より原理的な問題にする
- **distractor の罠変更**: 主語すり替え型→数値逆転型など、罠の手法を変える

#### 問題提示のフォーマット

```
問題 3（seed: 2015h27a Q15 の変形）
ある企業がシステムの二重化を検討している。2台のサーバで同一の処理を並行実行し、
結果を照合する方式を採用した場合の記述として、誤っているものはどれか。

A. 障害検出能力が高い
B. コンパレータで処理結果を比較する
C. 障害時は故障側を切り離して縮退運転が可能
D. 待機系は平常時にバッチ処理を行う
```

- **問題番号**と**seed の出典**を必ず表示する（「この問題は実際の過去問を元にしている」という権威を示す）
- 4択形式（ア/イ/ウ/エ or A/B/C/D）を維持

#### 問題数

seed の数と learner の反応に応じて柔軟に決める:
- seed が豊富（10問以上）: **30〜50問** 程度を目安に
- seed が少ない（5問未満）: **15〜25問** 程度
- learner が「もう十分」と言ったら打ち切りOK

### Step 5: メタフィードバック（毎問、これが Layer 2 の本体）

learner が答えたら、以下を**毎問**返す:

#### 5a. 正誤と正答の理由（1〜2行）

```
正解はDです。Dはデュプレックスシステム（コールドスタンバイ）の特徴であり、
デュアルシステムでは2台とも常に同じ処理を実行するため「待機系が別の仕事をする」概念がありません。
```

#### 5b. 自己説明の要求

```
あなたがこの選択肢を選んだ理由を一言で教えてください。
```

learner の自己説明を受け取ったら、それに対してコメントする。正しい推論プロセスなら「その思考は正しい」と強化し、間違った推論なら「そこが罠で、実際は〜」と修正する。

#### 5c. 罠分析

各 distractor がどんな罠の手法を使っているかを指摘する:

```
罠分析:
- A「障害検出能力が高い」→ 正しい記述。照合機構があるため障害検出に強い
- B「コンパレータで比較」→ 正しい記述
- C「縮退運転が可能」→ 正しい記述（フェールソフト）
- D「待機系でバッチ処理」→ ★罠: デュプレックスの特徴をデュアルに混ぜている（概念のすり替え型）
```

#### 5d. キュー抽出

```
この問題のキュー: 「同一の処理を並行実行」「結果を照合」→ デュアルシステムと即座に特定できる。
この2つの語句が揃っていたら、デュプレックスではない。
```

#### 5e. seed 過去問への参照（簡潔に）

```
元の過去問（2015h27a Q15）では「デュアルシステムの説明として正しいものはどれか」と正面から問うていました。
本問はそれを「誤っているものはどれか」に変形し、distractor にデュプレックスの特徴を混入させた形です。
```

### Step 6: leaf クリアと遷移

learner が十分な問題数をこなしたら（または自ら「次へ」と言ったら）:

1. **エピソードまとめ**: この leaf で見たパターンを 3〜5 行で要約

```
この leaf の出題パターンまとめ:
- デュアル vs デュプレックスの混同を突く問題が最頻出
- 「コンパレータ」「照合」がキーワードとして狙われる
- 障害時動作（縮退運転 / フェールオーバー）を入れ替える罠が定番
```

2. **次の leaf を提案**:

```bash
python3 /Users/tskymmt/git/aitee_training/.claude/skills/textbook-dispenser/scripts/dispense.py neighbors <current_leaf_id>
```

> 「次は ch04_03_02『デュプレックスシステム』の SFT に進みますか？」

consent-gated。learner の同意で次に進む。

## 守るべきこと

- **問題は1問ずつ**: 複数同時提示しない
- **毎問メタフィードバック**: 正誤だけで終わらない。自己説明→罠分析→キュー抽出を毎回入れる
- **seed の出典を明示**: 変形元の過去問（年度・問番号）を表示する
- **過去問そのままは出さない**: 必ず何かを変形する。全く同じ問題は seed の丸暗記を誘発する
- **leaf 単位で完結**: 1 leaf を始めたら途中で飛ばない
- **遷移は learner consent**: 「進みますか？」を必ず聞く
- **textbook_structure.json / past_exam/*.json は直接読まない**: 必ず各 dispenser を経由する

## してはいけないこと

- **過去問をそのまま出す**: NG。必ず変形する。seed はあくまでコンテクスト
- **フィードバックを正誤だけで済ます**: NG。メタフィードバック（自己説明・罠分析・キュー抽出）が Layer 2 の本体
- **新しい知識を教えようとする**: NG。それは Layer 1 の仕事。Layer 2 は既知の知識の取り出し方を訓練する場。learner が概念を知らないようなら Layer 1 に誘導する
- **seed が少ないからといってスキップする**: NG。seed が1問でもあれば5〜10の変形は作れる

## なぜこの構造か

過去問をそのまま解かせると N 問の丸暗記（オーバーフィッティング）で終わる。過去問は「試験がこの概念をどう問うか」の分布を定義する seed であり、その分布から変形問題を大量生成することで、分布全体を走破する汎化性能が身につく。これは ML の data augmentation と同じ原理。

メタフィードバック（自己説明・罠分析・キュー抽出）は、問題を解くだけでは訓練されない「試験の読み方」を育てる。正答を知るだけでなく、「なぜその罠に引っかかるのか」「どのキーワードが手がかりだったか」を言語化させることで、次の未知の問題でも同じパターンを見抜けるようになる。
