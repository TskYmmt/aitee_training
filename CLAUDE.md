# aitee_training

## プロジェクトの目的

日本の**応用情報技術者試験（AP）**の合格に向けて、暗記や概念理解のための反復トレーニングなど、主にトレーニング的な部分を効率よく鍛えるツールを細かく開発していくプロジェクト。

## マニフェスト

### 俺たちが作るものは、既存の過去問トレーニングの焼き直しではない。

- **過去問はあくまで参照物**であり、目的ではない。教科書を読んで理解した後に、その知識をあらゆる角度から問われても引き出せる力 ― "知識の検索力" を鍛えることが本質。
- 単なる「問題を解いて正解を覚える」サイクルではなく、**抽象的な概念理解を多面的に強化する**トレーニングを設計する。
- 世の中にありふれた学習メソッドを常に疑う。過去問周回、一問一答、暗記カード ― それらが本当にベストか？ **俺たちの方がもっと効果的なものを作れる**という前提で設計する。
- 目指すのは「試験に受かるだけの知識」ではなく、**試験にも適用でき、かつ実務でも使える本物の知識**。
- 斬新であること自体が目的ではないが、既存手法より明確に優れた効果が見込めるなら、躊躇なく新しいアプローチを採用する。

## コンセプト

本プロジェクトは、**LLMの訓練パイプラインを設計メタファ**として、受験者の学習システムを構築する。受験者を1つのニューラルネットワークとみなし、各段階で「何が」「どのように」更新されているかを意識的に設計する。

### 3レイヤ構造

| Layer | 内容 | 狙い |
|---|---|---|
| **Layer 1: Pre-training** | 丸暗記の効率的反復（離散事実＋概念核の両方） | 脳内に retrievable な表現を作る |
| **Layer 2: SFT** | 試験フォーマットへの出力適応 | 持っている知識を試験特有の問い方で正しく出力できるようにする |
| **Layer 3: Maintenance** | 既習 leaf の drift 防止と強化 | 長期保持。Layer 1/2 の成果物をスケジュールして再暴露 |

**決定的な含意**: 過去問はあくまで Layer 2 のデータソースであって、**Layer 1 の corpus ではない**。世間の試験対策は Layer 1 をスキップして Layer 2 に全振りしているため、「過去問は解けるが聞き方を変えられると死ぬ」現象が起きる。これはLLMで言えば過学習SFTモデルそのもの。本プロジェクトは Layer 1 を主軸に据えることで、試験にも実務にも使える「本物の知識」を育てる。

### Layer 1: Pre-training（初版実装済み）

テキストの leaf（サブセクション）単位で進行。1 leaf = 1 エピソード:
1. 概念の密度高い説明
2. 隣接概念への予告（bridge）
3. 10〜20問の engagement loop（1問ずつ提示、4択、ジャンル混在）
4. 毎問フィードバック（正誤＋なぜ＋他選択肢の正体＋ニーモニック）
5. まとめ → consent-gated で次の leaf へ

問題は LLM がテキスト知識から新規生成する。過去問は使わない。「丸暗記」には離散事実（ポート番号、略語）と概念核（LRUの原理、正規化の論理）の2種があり、前者は機械的リコール反復、後者は生成的反復（自己説明、別文脈転用）と、反復のモダリティを使い分ける。

実装: `.claude/skills/pre-training/` + `.claude/skills/textbook-dispenser/`

#### 専用トレーニングメニュー（計算・手続き系 leaf 向け）

319 leaves の大半は宣言的知識であり、上記の MCQ エピソードで十分。ただし**計算・手続き的知識を要する leaf 群**は MCQ だけでは不十分 — 4択で正解を選べても「自分で導出する力」が育たない。過去問分析の結果、全2640問中 約40% が計算・手続き系の問題であり、以下のセクションに集中している。

これらの leaf に対しては、テーマ別に**専用のトレーニングメニュー**を設計する。抽象的な汎用フレームワークではなく、テーマごとに最適な入力形式・視覚表現・フィードバック・進捗指標を具体的に設計する方針（具体から入ってパターンが見えてから抽象化）。

**設計原則**: LLM が記号の意味を獲得したのは特別な grounding メカニズムではなく、**膨大な文脈での反復露出（数の暴力）**だった。人間にも同じ原理を適用する — 特殊なモードを作るより、**同じ概念を大量の異なる角度から繰り返し処理させる**ことで意味の即時発火状態を作る。ただし人間の学習時間は有限なので、角度のバリエーションを最大化して効率的に達成する。

##### 実装方針
- **Menu 1（論理・集合演算 反射ドリル）から作る**。最もシンプルで、Claude チャットインターフェースでの成立性を検証できる原型
- 成立したら残りのメニューを順次作る。作る過程で共通パターンが見えたらその時初めて抽象化を検討する
- 各メニューの実装は**テーマに最適化した個別スキル**として作る。汎用フレームワークに無理に合わせない

### Layer 2: SFT（設計確定、実装着手前）

Layer 1 で蓄えた知識を、**試験という特殊な取り出し環境で正しく出力する訓練**。新しい知識を入れる場ではない。

**進行単位**: Layer 1 と同じく **leaf 単位**。1 leaf をクリアしたら次に進む。

**Entry ゲート**: 理想は Layer 1 を全体完了した後に開始。ただし厳密強制はしない。ユーザー希望があれば部分的にも対応する。スキルは理想状態を内部に保持する。

**核心: 過去問は「出題空間の定義」であって直接解かせるデータではない**

- leaf X に紐づく過去問 N 問は、「試験がこの leaf をどう問うか」の分布を定義する seed
- seed をそのまま解かせたら N 点の丸暗記 → オーバーフィッティング
- 代わりに seed をコンテクストに入れ、LLM が**同じ分布の中から変形問題を大量生成**する
- 変形: 数値変更、シナリオ差替、正答/distractor 入替、問い方の逆転、隣接概念との合成
- これは ML の data augmentation そのもの。real data から分布を保ったまま多様性を増やし、汎化性能を上げる

**1 leaf あたりの問題数**: seed × 5〜10倍 = 数十問。ペースは Layer 1 より速い（1問 3〜5分）。量の節約はしない、構造の質で勝負する。

**Engagement enforcement の所在はフィードバック側**:
- 毎問、learner に「なぜその選択肢を選んだか」を述べさせる（自己説明）
- 正答時も「他の選択肢のどこが罠か」を分析させる（distractor 分析）
- 「問題文のどの語が正答への手がかりだったか」を言語化させる（キュー抽出）
- これらが Layer 2 の本体。問題を解くこと自体は手段であり、メタ認知の訓練が目的

**空間定義の実装方針**: まず汎用プロンプト1本（seed 過去問を受け取り変形問題を生成する共通指示）で試す。seed 自体が leaf ごとの出題特性を暗黙に運ぶので、多くの leaf はこれで足りるはず。distractor の質が落ちる leaf カテゴリが見つかった場合のみ、カテゴリ別ガイド（計算系 / 用語識別系 / プロセス系 / 法規系 等、5〜8種）を追加する。leaf 個別のデータ作成は最終手段。

**Layer 2 が訓練する固有スキル（Layer 1 では訓練されないもの）**:
- 試験問題の定型表現の解釈（「最も適切な」「誤っているものはどれか」等の意味）
- Distractor immunity（もっともらしいが微妙にズレた選択肢への耐性）
- キュー → 想起パスの強化（問題文の手がかり語から知識領域を瞬時に呼び出す）
- Cross-leaf integration（複数 leaf にまたがる知識を要する問題への対応）
- 試験のメタ知識（頻出パターン、時間配分、捨てる判断、記述での部分点）

### Layer 3: Maintenance（未着手）

Layer 1/2 の成果物（習得済み leaf）に対して上から作用するメタ操作レイヤ。drift 防止と consolidation を兼ねる。設計は Layer 2 が動き始めてから詰める。

### 人間脳への3つの補正

LLM訓練の枠組みをそのまま脳に当てはめることはできない。3つの質的な違いに対応する補正が必須である。

**補正1: フェーズは離散ではなくスペクトラム、そして互いに干渉する**
LLMはコード上でフェーズを切替できるが、脳では同じ活動の中に複数のフェーズが同時に弱く走っており、配分を間違えると干渉する（問題演習モードに入りすぎると概念吸収モードが起動しにくくなる等）。
→ フェーズを switch するのではなく、**mix ratio を連続制御する**ミキサー型設計にする。

**補正2: インプット自体がアクティブでなければ実効学習率はゼロ**
LLMは入力を流せば自動で勾配が流れるが、脳は「目が滑る」現象が示す通り、**露出 ≠ 学習**。engagement intensity がゼロなら η=0 で、何時間読んでも重み更新ゼロ。
→ pre-training フェーズですら受動読書は破綻。**すべての入力イベントに、強制的に forward pass を起こす仕掛け**（予測→検証、空欄補完、自己説明、別文脈転用、教え返し、対比による違和感生成など）を埋め込む。

**補正3: 重みは時間で減衰する — maintenance は first-class なフェーズ**
LLMには「維持する」概念が存在しないが、脳は訓練停止の瞬間から drift が始まる。
→ Phase 1〜4 と並列で常時走る**第5のフェーズとしての maintenance loop** を持つ。しかも retrieval は単なる refresh ではなく、それ自体が weight update を引き起こす（testing effect）ため、**maintenance は preservation かつ consolidation**。受験当日まで止められないため、長期的には dominant mode となる。

### 3機構のオーケストレータ

以上の補正から、システムは次の3機構を同時に制御する**オーケストレータ**として設計される:

1. **Mix ratio controller** — 各瞬間における Layer 1 / Layer 2 / Layer 3 の配合比を動的制御する
2. **Engagement enforcer** — あらゆる入力を「脳内で強制的に forward pass が走る形」に変換する仕掛けの集合
3. **Maintenance loop** — 既習の重みの drift を防ぐために常時走る並列プロセス（consolidation も兼ねる）

### 「受験者が考えなくていい」の技術的意味

上記3機構は**システム側が担う**。受験者から見れば「ただ提示される刺激に反応しているだけ」だが、内部では3機構が連動して常に最適な勾配を流し続けている状態になる。受験者は「勉強法を考える」という認知資源を一切使わず、**純粋に理解と想起そのものに全リソースを振れる**。これが本プロジェクトの最終的な受験者体験の設計目標である。
